#{msgs.title} - Application Programming Interface

Introduction

#{msgs.title} manages naming elements for FRIB's Naming Convention. Data in #{msgs.title} can be accessed programmatically through a RESTful Application Programming Interface (API), which is described in this document.

Version

The #{msgs.title} API follows Semantic Versioning for its version numbers. The current API version number is #{msgs.apiversion}.

Base URI

The base URI for Resource Identifiers, on this server, is #{restRoot}

Caching

Currently the #{msgs.title} API does not support caching. However, future versions of the API will support caching mechanisms such as Conditional GETs and Expiration Times.

Authentication and Authorization

This version of the API does not restrict access to its resources. However, future versions will require client applications to authenticate themselves to access this API. So please be prepared to update you application. Note: Authentication mechanisms of Hour Log API will be changed in future. So please be prepared to update you application.

Encryption

Hour Log API forces the use of HTTPS protocol. If HTTP is used, Hour Log will redirect it over to HTTPS. Note that the public key of the Hour Log development and QA server may not be signed by trusted Certificate Authorities.

Error Codes and Responses

The Hour Log API attempts to return appropriate HTTP Status Codes for each request.
